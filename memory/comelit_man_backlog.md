## Schema

| Column | Meaning |
|---|---|
| ID | `BL-NNN`, stable across sweeps |
| Title | One-line description |
| Sev | `Critical` (breaks user / leaks creds) / `High` / `Medium` / `Low` |
| Gate | Tier this unblocks: `bronze` / `silver` / `gold` / `platinum` / `none` (hygiene) |
| Files | Touched files (or `TBD` until sweep confirms) |
| Locked? | `YES` if any LOCKED file is touched (`door.py`, `video_call.py`); requires owner approval before any change |
| Effort | `S` / `M` / `L` |
| Status | `Triage pending` (skeleton) / `Confirmed` / `In progress` / `Done` / `Won't fix` / `Blocked` |
| Linked rule | Rule slug or beyond-scale dimension (e.g. `bronze:runtime-data`, `beyond:A`) |

Severity, Gate, Effort are set/refined during Sweep 5 triage. At Sweep 0 they reflect the plan author's prior estimate, marked `Status: Triage pending` so nothing is treated as confirmed.
